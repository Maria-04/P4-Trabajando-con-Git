Push & Pull 3

Este ejercicio comienza con una brevísima explicación de lo que es trabajar en remoto, que es transferir datos de un repositorio y hacia otros repositorios.

El objetivo de este ejercicio es aprender a usar el comando de git, "git fetch". Aunque para eso primero tenemos que saber lo que hace 'fetch'.

Fetch

¿Qué hace?

  • Baja los commits que están en el repositorio remoto y que no están en nuestro repoositorio local.
  • Actualiza a donde apuntan nuestras ramas remotas.
  • Sincroniza nuestra representación local del repositorio remoto con el verdadero estado de este último.

Como hemos visto en ejercicios anteriores, las ramas remotas reflejan el estado de los repositorios remotos desde la última vez que nos dirijimos a ellos; pues bien, git fetch es la manera con la que nos dirijimos a ellos, ya que git fetch se comunica con el repositorio a través de Internet (usando un protocolo http:// o git://)

¿Qué no hace?

NO modifica tu estado local, NO actualiza tu rama principal (main,master...) y NO cambia nada sobre cómo se ve tu sistema de archivos en el momento en el que haces 'git fetch'.

Esto es importante tenerlo claro porque podemos pensar que git fetch hará que nuestro estado local refleje el estado del remoto. Git fetch puede descargar los datos necesarios para eso, pero no cambia ninguno de nuestros archivos locales. En resumen, hay que pensar en git fetch como 'un proceso de descarga'.
